Overview

🚀

Push Routes

Inject new Cribl routes above the default catch-all, targeting a specific pipeline and destination, across one or more worker groups.

🗑️

Add ELK Roles & Cribl Routes

Create ELK security roles and role-mappings in nonprod and prod, and inject Cribl routes and destinations — in a single coordinated operation.

📋

Single or Bulk

Process one app by ID + name, or upload a .txt file with multiple apps (appid, appname per line).

🔒

Safe by Default

Dry-run is checked by default. Protected workspaces require an explicit "Allow production writes" override before any changes are made.

Quick Start

  1. 1

    Open the app

    Click Launch App above. The UI will load in a new tab.

  2. 2

    Select a workspace and worker group(s)

    Pick the target Cribl workspace from the dropdown, then choose one or more worker groups. If the workspace is marked as protected, check Allow production writes.

  3. 3

    Enter app details

    Single mode: provide App ID and App Name.
    Bulk mode: upload a .txt file — one appid, appname pair per line. Lines starting with # are ignored.

    # example apps.txt
APP001, Payments Service
APP002, Auth Gateway
# APP003, Deprecated App   <-- skipped
  4. 4

    Verify with Dry Run first

    Leave Dry Run checked (default). Review the diff output on the right. No changes are written until you uncheck it and re-run.

  5. 5

    Run

    Click Run cribl-pusher. Output and exit code appear in the right panel. A green banner means success (exit 0).

Field Reference

📡 Tab 1 — Cribl Pusher
  • Cribl URL optional Overrides config.json base URL
  • Workspace required Target Cribl workspace
  • Worker Group(s) required One or more worker groups; runs sequentially
  • Region required azn or azs
  • App ID / App Name required Single mode only
  • App list file required Bulk mode — .txt, one entry/line
  • Dry Run flag Simulate only — no writes (default: on)
  • Skip SSL flag Disable TLS verification
  • Bearer Token optional Overrides config.json credentials
  • Route Group ID optional Target a specific route group
  • Min Existing Routes optional Safety guard — abort if below threshold
  • Snapshot Dir optional Save before/after JSON snapshots
⚡ Tab 2 — ELK Roles + Cribl Routes
  • App ID / App Name required Single mode only
  • ELK Nonprod URL required Unless Skip ELK is checked
  • ELK Prod URL required Unless Skip ELK is checked
  • ELK Token / User+Pass required Token takes priority over user/pass
  • Cribl Workspace required Unless Skip Cribl is checked
  • Order flag ELK first or Cribl first
  • Skip ELK flag Only push ELK roles, skip Cribl
  • Skip Cribl flag Only push Cribl routes, skip ELK
  • Dry Run flag Simulate only (default: on)
  • Allow Prod Writes flag Required for protected workspaces

Credential Resolution Order

ℹ️ Credentials are resolved in this order: UI field → environment variable → config.json. Leave UI fields blank to fall back to config.json. 
# Environment variable fallbacks
export CRIBL_TOKEN=your-token
export CRIBL_USERNAME=admin
export CRIBL_PASSWORD=secret

Safety Notes

⚠️ Dry Run is on by default. Always review the diff output before unchecking it. Routes are inserted above the first final: true route — verify the position is correct in dry-run output.
⚠️ Protected workspaces require the Allow production writes checkbox to be checked. The run will be rejected otherwise.
